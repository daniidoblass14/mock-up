RESUMEN EJECUTIVO
================================================================================

AutoLytix es una aplicación SaaS de gestión de flotas de vehículos desarrollada 
como demo UI completa sin backend real. La aplicación permite gestionar 
vehículos, mantenimientos, visualizar calendarios, analizar costes y obtener 
recomendaciones orientativas.

ESTADO DEL PROYECTO:
- Frontend completo con todas las funcionalidades core
- Persistencia local mediante localStorage
- Datos mock para demostración
- Tema claro/oscuro implementado
- Responsive design en todas las pantallas
- Sin backend - todo funciona en memoria/localStorage

TECNOLOGÍAS PRINCIPALES:
- React 18.2.0 + TypeScript
- Vite 5.0.8
- Tailwind CSS 3.3.6
- React Router 6.20.0
- Recharts 3.7.0 (gráficas)
- Lucide React 0.294.0 (iconos)

LO QUE ESTÁ HECHO - FUNCIONALIDADES IMPLEMENTADAS
================================================================================

1. LOGIN (/login)
   - Formulario de inicio de sesión
   - Validación básica
   - Redirección al dashboard

2. DASHBOARD (/dashboard)
   - Métricas principales: vehículos operativos, mantenimientos próximos/vencidos, coste del mes
   - Alertas rápidas de mantenimientos vencidos/próximos
   - Banner de onboarding (se puede cerrar)
   - Acciones rápidas para registrar vehículo y ver mantenimientos

3. VEHÍCULOS (/vehiculos)
   - Lista completa de vehículos con tabla
   - Paginación client-side (10/20/50 por página)
   - Búsqueda por modelo, matrícula, tipo, VIN
   - Filtros por estado (Todos, Al día, Próximo, Vencido)
   - Acciones: Ver detalle, Editar, Eliminar
   - Formulario "Añadir Vehículo" con validaciones completas:
     * Campos obligatorios: Modelo, Tipo, Año, Matrícula, Kilometraje, Estado
     * Validación de formato de matrícula (0000-AAA)
     * Validación de duplicados (marca+modelo+matrícula)
     * Errores inline debajo de cada campo
     * Botón deshabilitado si hay errores
   - 32 vehículos mock predefinidos

4. DETALLE VEHÍCULO (/vehiculos/:id)
   - Vista dedicada (no modal)
   - Breadcrumb y botón volver
   - Imagen placeholder
   - Grid con todos los datos del vehículo
   - Historial completo de mantenimientos del vehículo

5. MANTENIMIENTOS (/mantenimientos)
   - Tabs: Próximos, Vencidos, Completados
   - Búsqueda por vehículo o tipo
   - Métricas: vencidos, próximos 7 días, costo del mes
   - Tabla completa con todas las acciones
   - Formulario "Añadir Mantenimiento" con validaciones:
     * Campos obligatorios: Vehículo, Tipo, Coste, Fecha objetivo, Km objetivo
     * Errores inline
     * Botón deshabilitado si hay errores
   - Sincronización automática: al crear mantenimiento, aparece en calendario

6. DETALLE MANTENIMIENTO (/mantenimientos/:id)
   - Vista dedicada (no modal)
   - Breadcrumb y botón volver
   - Resumen completo con cards informativos
   - Información del vehículo relacionado
   - Notas si existen

7. CALENDARIO (/calendario)
   - Vista mensual con grid de calendario
   - Navegación: mes anterior/siguiente, botón "Hoy"
   - Filtros: por vehículo, por tipo de mantenimiento
   - Búsqueda por tarea o vehículo
   - Vista semanal en sidebar
   - Muestra hasta 2 tareas por día con indicador "+X más"
   - Sincronización automática sin recargar página

8. GRÁFICAS (/graficas)
   - Tabs: Por vehículo / Flota
   - Tab "Por vehículo":
     * Selector de vehículo
     * Métricas: Total gastado, Gasto anual medio, Últimos 12 meses
     * Gráfica de costes acumulados (AreaChart)
     * Sistema de recomendaciones:
       - Niveles: Mantener, Vigilar, Valorar cambio, Insuficientes
       - Basado en antigüedad, ratio de gasto, kilometraje
   - Tab "Flota":
     * KPIs: Gasto total, Año actual vs anterior, Nº vehículos
     * Gráfica de gasto por años
   - Empty state cuando no hay datos

9. PERFIL (/perfil)
   - Formulario de perfil de usuario
   - Cambio de contraseña
   - Configuración de preferencias
   - Gestión de foto de perfil

COMPONENTES REUTILIZABLES:
- Modal: reutilizable con diferentes tamaños
- CustomSelect: select personalizado con estilos consistentes
- ConfirmDialog: diálogo de confirmación para acciones destructivas
- Header: búsqueda global, notificaciones, toggle tema
- Sidebar: navegación principal con estado activo
- Layout: layout principal con Sidebar + Header

SERVICIOS (MOCK):
- VehiculosService: CRUD completo de vehículos
- MantenimientosService: CRUD completo de mantenimientos
- CalendarioService: CRUD completo de tareas de calendario

CONTEXTOS:
- AppContext: estado global (vehículos, mantenimientos, tareas)
- ThemeContext: gestión de tema claro/oscuro con persistencia
- ToastContext: sistema de notificaciones

UTILIDADES:
- currency.ts: formateo de moneda y números
- matriculaMask.ts: validación y formato de matrículas
- recomendacion.ts: lógica de recomendaciones de vehículos

PERSISTENCIA:
- localStorage para vehículos y mantenimientos
- Carga automática de datos mock si no hay datos guardados
- Persistencia automática al crear/actualizar/eliminar

DATOS MOCK:
- 32 vehículos con perfiles variados (nuevos, estables, problemáticos, veteranos)
- 65+ mantenimientos distribuidos en años 2021-2025
- Costes variados (€38 - €900)
- 22 tipos diferentes de mantenimiento

VALIDACIONES IMPLEMENTADAS:
- Formulario vehículo: todos los campos obligatorios, formato matrícula, duplicados
- Formulario mantenimiento: todos los campos obligatorios, números válidos
- Errores inline en tiempo real
- Botones deshabilitados si hay errores

CARACTERÍSTICAS DESTACADAS:
1. Sincronización automática: mantenimientos → calendario
2. Paginación client-side funcional
3. Navegación por rutas (no modales para detalles)
4. Sistema de recomendaciones basado en datos reales
5. Tema claro/oscuro con persistencia
6. Búsqueda global en header
7. Gráficas interactivas con Recharts
